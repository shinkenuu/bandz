/api/music-genres/

Read-only

/api/places/

Pass in an place_id or a Place obj so I can read the place_id form it, query Google and return what GOOGLE told me

/api/users/

Create a user before any fa, band or host

/api/fas/

Has a User and can relate to Event through /api/event-public/

/api/bands/

Has a User and can create propositions via /api/propositions/

/api/hosts/

Has a User and can create events at /api/events/ and accept propositions at /api/propositions/

/api/events/

The fields 'confirmed' and 'band' are read-only, populated automatically based on what happens at propositions

/api/propositions/

Must only be created by hosts. Once created, only modify the 'confirmed', ' message' and 'price' fields. Once confirmed, the associated event is confirmed and the winner band gets into the event instance

/api/event-public/

Opinion of fas about an event
